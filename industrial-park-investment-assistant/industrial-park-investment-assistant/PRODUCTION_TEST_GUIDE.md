## 📋 测试清单

### 测试1：缓存命中验证（最重要！）

**测试步骤**：

```
第1步：首次查询（测量MCP调用）
  问：「示例产业园A还有没有500平的房源？」
  观察：
    ✅ AI是否调用了MCP工具？（应调用）
    ✅ 响应时间是否约2-3秒？
    ✅ AI是否提示「已保存到缓存」？

第2步：重复查询（测试缓存命中）
  问：「租金多少？」（等待10秒后问）
  观察：
    ✅ AI是否【没有】调用MCP工具？（应使用缓存）
    ✅ 响应时间是否 <0.5秒？
    ✅ AI是否提示「使用缓存数据」？

第3步：缓存过期测试（可选）
  问：「刷新数据，重新查询」（等待30分钟后）
  观察：
    ✅ AI是否重新调用MCP？
    ✅ 缓存是否更新？
```

**预期结果**：

| 指标 | 首次查询 | 重复查询 | 提升倍数 |
|------|---------|---------|---------|
| MCP调用次数 | 1次 | 0次 | 减少100% |
| 响应时间 | 2-3秒 | <0.5秒 | **4-6倍** |
| 数据来源 | MCP | 本地缓存 | - |

## ⚠️ 常见问题排查

### 问题1：AI还是调用了MCP（缓存未命中）

**可能原因**：
1. 缓存文件不存在或已过期（>30分钟）
2. AI没有检查本地缓存
3. 用户说了「刷新数据」强制刷新

**排查步骤**：
```bash
# 检查缓存文件是否存在
ls -la ~/.workbuddy/workspace/investment-assistant/projects/meilan-center/data/
ls -la ~/.workbuddy/workspace/investment-assistant/projects/meilan-center/cache/

# 检查缓存时间戳
stat ~/.workbuddy/workspace/investment-assistant/projects/meilan-center/data/房源销控表.json
```

---

### 问题2：响应时间还是很长（>3秒）

**可能原因**：
1. AI没有遵循并发调用规则
2. AI尝试了多个文档才找到数据
3. 网络延迟或MCP服务响应慢

**排查步骤**：
1. 观察AI的tool calls：是否并发？还是串行？
2. 检查AI是否直接定位到正确的文档？
3. 查看PERFORMANCE.md中的"MCP调用延迟"章节

---

### 问题3：缓存数据过期（数据已更新，但AI使用旧数据）

**解决方案**：
1. 用户说：「刷新数据」「重新读取」
2. AI强制刷新缓存，重新调用MCP
3. 或者等待30分钟后，缓存自动过期
